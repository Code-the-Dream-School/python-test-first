Book Titles

Topics

  • classes and objects
  • instance variables
  • getter and setter methods
  • strings

Notes

Book Titles in English obey some strange capitalization rules. For example, "and" is lowercase in "War and Peace". This test attempts to make sense of some of those rules.

In this lesson, you should use a private instance variable such as __title in your Book class. Private instance variables begin with two underscores, so as to discourage people from accessing them outside the class, except through getters and setters. The setter function should capitalize the book title appropriately before storing it in the private instance variable. Remember that the methods of a Python class always take self as the first argument, BUT you don't pass this argument when calling the function.

You can reuse your titleize function from the simon_says exercise -- but change it to handle self.
